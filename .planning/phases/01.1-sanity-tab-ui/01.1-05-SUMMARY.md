## Why It Got Built This Way

### Task 1 verification: vanilla `drawBorder` exists

Before applying Task 3's listbox border fix, ran a static read on `ProjectZomboid/media/lua/client/ISUI/ISScrollingListBox.lua` to confirm the planning context's claim that `drawBorder` is a vanilla flag.

**FINDING — VERIFIED:**
- Line 484-485: `if self.drawBorder then self:drawRectBorder(0, -self:getYScroll(), self.width, self.height, self.borderColor.a, self.borderColor.r, self.borderColor.g, self.borderColor.b) end`
- Line 716: `o.drawBorder = false` (default)
- Line 707: `o.borderColor = {r=0.4, g=0.4, b=0.4, a=0.9}` (default)

**LOCKED CHOICE:** Use `self.eventLog.drawBorder = true` flag. Manual drawRectBorder fallback NOT required.

Reasoning: one-line change in `:createChildren` is simpler and more idiomatic than a manual border render in `:render`; vanilla draws at `(0, -getYScroll())` correctly anchored at the listbox's coordinate origin (manual rendering would have to compute panel-relative coords); default `borderColor = {r=0.4, g=0.4, b=0.4, a=0.9}` already matches the panel-border style.
