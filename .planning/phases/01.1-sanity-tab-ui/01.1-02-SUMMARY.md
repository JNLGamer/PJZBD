## Logger Contract

```lua
function SanityTraits.log(category, text, delta)
    -- category : "kill" | "stage" | "trait" | "recovery"   — invalid prints WARN, returns
    -- text     : English string (D-23 hardcoded)
    -- delta    : signed integer or nil (nil for stage/trait events without numeric)
    -- side effect: prepends {time, category, delta, text, icon=nil} to md.SanityTraits.log
    --             and evicts oldest if #log > LOG_MAX_ENTRIES
end
```

**Whitelist (D-13):** `kill`, `stage`, `trait`, `recovery`. `kill` is the only category fired in Phase 01.1; the other three are accepted now so Phase 2 / 3 / 5 can call the logger without modifying it.

**Entry shape (D-14):**

```lua
{
    time     = getTimestampMs(),  -- ms since epoch; deterministic across saves
    category = "kill" | "stage" | "trait" | "recovery",
    delta    = -10 | -30 | nil,   -- signed integer or nil
    text     = "Zombie killed" | "Survivor killed" | ...,
    icon     = nil,               -- v1 hardcodes nil
}
```

**Insert + FIFO (D-12, D-14):**

```lua
table.insert(md.SanityTraits.log, 1, entry)         -- newest at top
while #md.SanityTraits.log > SanityTraits.LOG_MAX_ENTRIES do
    table.remove(md.SanityTraits.log)               -- no second arg = removes tail (oldest)
end
```

## Decisions Made

- **Logger lives in `4_SanityTraits_Panel.lua` (not `4_Logger.lua` + `5_Panel.lua`).** Plan 03 extends the same file with the `SanityPanel` class. The logger and the panel share `md.SanityTraits.log` as their single source of truth — co-locating them keeps the read/write contract obvious.
- **Whitelist via local upvalue table.** `VALID_LOG_CATEGORIES = {kill=true, stage=true, trait=true, recovery=true}` keeps category validation O(1) and trivially extensible. Fails closed: unknown category prints WARN with LOG_TAG and returns without writing.
- **FIFO eviction loop uses `while`, not `if`.** Defensive against any future bulk-insert that pushes the array over by more than one element. Cost is negligible — typical operation runs the loop body zero times.
- **Console receipt `[SanityTraits] log[<cat>] <text> (delta=<n>)`.** Standardized format using LOG_TAG so console.txt grep stays clean. The `(delta=<n>)` clause is omitted when `delta` is nil (stage/trait events that don't carry a number).
- **Plan 1 console output preserved.** The existing `Zombie killed. Sanity: X -> Y` / `Survivor killed. Sanity: X -> Y` print lines are not replaced — the new logger call appears below them. Phase 1 UAT row that depends on those exact strings stays green.
