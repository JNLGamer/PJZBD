## Decisions Made

- **Trait icon source uses `TraitFactory.getTrait(id):getTexture()` instead of the D-18 path-based form.** Vanilla's `media/ui/Traits/` directory holds only 16 profession-related PNGs. None of our negative traits (cowardly, hemophobic, slowhealer, weakstomach, insomniac, disorganized) ship a `Trait_<id>.png`. The Java-side `TraitFactory.Trait` object holds the authoritative texture; the procedural fallback rect handles the case where even that returns nil.
- **No tab-strip width adjustment.** Vanilla's `tabTotalWidth` is a local variable inside vanilla `:createChildren` ã inaccessible after the wrap returns. ISTabPanel handles overflow via built-in scroll arrows. Default behavior shipped; UAT will tell us whether visible truncation matters.
- **Empty-state listbox uses a real `addItem('No events yet.', nil, nil)`.** A separate ISLabel widget would have required additional layout state and fight ISScrollingListBox's auto-sizing; using `addItem` with a nil backing keeps the empty-state visually consistent with populated entries (same font, same padding).
- **Numeric readout reserves 110px right-side slot before computing bar width.** Keeps the `999 / 1000 (100%)` text fully visible at any panel width without truncation. The reservation is a fixed pixel value rather than a measured one because `getTextManager():MeasureStringX` is heavier per call than a const lookup, and the format string's max width is bounded.
- **Cache pattern uses `-1` sentinel in `:new`.** Both `lastLogCount` and `lastAppliedCount` initialize to `-1` (not `0`) so the first `:render` frame always detects a "change" and populates. With `0` initial, the first frame after a fresh-character open with `#log == 0` would skip the rebuild and the empty-state placeholder would never render until the first kill.
- **Procedural fallback rect is drawn from `:render` on top of the ISImage's slot position.** When `useFallback` is true, the ISImage's `.texture` is nil so the widget itself contributes no visible pixels ã our procedural drawing on top fills the slot. This keeps the layout invariant (one cell per applied trait) regardless of whether TraitFactory has an icon.
- **Wrap is installed at file-load time (top-level code).** The `if not ISCharacterInfoWindow then ... else ... end` block runs once when `4_SanityTraits_Panel.lua` is loaded by the PZ mod loader; the `else` branch captures the original `:createChildren` and installs the wrap. The error branch only fires if vanilla loading failed (a defensive guard, not an expected path).

## Notes for Downstream Phases

### Phase 2 (Stage Transitions)

When a sanity threshold is crossed (e.g. character drops from 751 ã 749, entering "shaken"):

```lua
-- Compute the new stage
local newStage = SanityTraits.computeStage(md.sanity)
local stageName = SanityTraits.STAGE_NAMES[newStage]

-- Record the transition in the log (the panel's listbox will rebuild on next render)
SanityTraits.log("stage", "Stage changed to " .. stageName, nil)

-- Apply the appropriate trait (e.g. base:cowardly at shaken)
table.insert(md.appliedTraits, {
    traitId        = "base:cowardly",
    appliedAtStage = newStage,
    appliedAtTime  = getTimestampMs(),
})
SanityTraits.log("trait", "Trait applied: " .. trait:getLabel(), nil)
player:getTraits():add("base:cowardly")
```

The panel's `refreshDebuffRow` will detect `#md.appliedTraits` changed on the next frame and populate the new debuff slot with the trait icon and tooltip ã no panel code changes needed.

### Phase 3 (Passive Decay & Recovery)

`SanityTraits.log("recovery", "Sanity recovered", +5)` and `SanityTraits.log("kill", "Passive decay", -1)` (or any other category) work without logger changes. Recovery deltas render with a `+` sign in the listbox (formatLogEntryDisplay handles signed-positive deltas).

### Phase 5 (Habit-based Addiction)

When the chosen addiction trait is selected at "depressed" stage:

```lua
table.insert(md.appliedTraits, {
    traitId        = chosenAddictionTraitId,
    appliedAtStage = "hollow",
    appliedAtTime  = getTimestampMs(),
})
```

If `TraitFactory.getTrait(chosenAddictionTraitId)` doesn't have a texture, the procedural fallback rect renders the last character of the trait ID. Custom texture support could be added in a polish phase by extending `getTraitTextureSafe` to consult an icon-override table on the SanityTraits namespace.
