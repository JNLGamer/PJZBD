# Phase 01.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
>
> **Reality check:** Project Zomboid Build 42 Lua mods have no test runner. The engine's UI/event system is tightly coupled to a live game state — there is no way to instantiate `ISCharacterInfoWindow` or fire `Events.OnZombieDead` outside a running PZ session. Every behavioral validation in this phase is **in-game UAT**. Static validations (string presence, table shape, function signature) are grep + code review.

## Sampling Rate

- **After every task commit:** Run the static grep-checks for that task's `<acceptance_criteria>`. Cost: ~1 second.
- **After every plan wave:** Reload the mod in-game (full PZ relaunch) and spot-check the visible behavior added in that wave. Cost: ~30 seconds per check.
- **Before phase verification:** Full in-game UAT table below must pass — every row green. Cost: ~3 minutes.
- **Max feedback latency:** ~30 seconds for static checks; ~3 minutes for full UAT pass. (No way to drop UAT cost lower without a real test runner.)

## Manual-Only Verifications

**All behavioral verifications in this phase are manual-only.** PZ engine APIs (UI rendering, event firing, ModData persistence, trait factory) cannot be exercised outside a running game session. The full UAT table above is the manual test plan.

| Behavior | Why Manual | Test Instructions |
|----------|------------|-------------------|
| Tab visible & non-crashing | Cannot instantiate `ISCharacterInfoWindow` outside live game | Launch PZ, start a game, open character window |
| Bar gradient renders correctly | Cannot capture screen output programmatically | Visual inspection, multiple sanity values via console |
| Tooltip on hover | Mouse hover events only fire under real input | Manual hover, visual readout |
| ModData persistence | `getModData()` semantics only resolve under live ModDataManager | Save → quit to main menu → reload save → verify values |
| FIFO log eviction at cap | `removeFirst` is broken in `ISScrollingListBox` (per RESEARCH Pitfall) — implementation must work around | Kill 60 zombies, count log entries, confirm oldest dropped |
| Trait icon rendering with `TraitFactory` | `TraitFactory` only populated under live `OnGameBoot` → cannot inspect outside game | Push known trait id via console, verify icon renders |
