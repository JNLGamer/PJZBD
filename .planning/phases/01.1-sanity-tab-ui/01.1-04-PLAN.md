**LOCKED DESIGN DECISIONS for this plan (call out so executor doesn't re-litigate):**

1. **Bar fill direction: BOTTOM-fill, gradient GREEN at TOP.** At sanity 1000 the bar is fully filled with the gradient (green visible at top, red visible at bottom). At sanity 500 only the bottom half shows gradient (yellow->red); the top half is dark track (the green portion is "drained"). At sanity 0 only the dark track shows. Justification: matches user's reference image semantics (smooth green->yellow->red gauge), matches thermometer metaphor, "losing sanity" intuitively shows the danger zone (red) holding while the safe zone (green) drains away.

2. **Gradient is ALWAYS rendered for the FULL bar height first; track is then drawn OVER the empty top portion.** Two-pass render: pass 1 = gradient bands across full bar; pass 2 = track-color rect over the top `(1-pct)*h` height to mask empty area. This guarantees the visible gradient at the bottom always matches the same colors at the same Y positions regardless of fill level (no rescaling, no "stretching").

3. **BAND_COUNT = 10.** Each band is a horizontal `drawRect` filling the bar's full width with a single color. Color is lerped using `bandPct = (band - 0.5) / BAND_COUNT` (band centroid in 0..1), where `bandPct = 0` is top of bar (green) and `bandPct = 1` is bottom of bar (red). Justification: 10 bands at 14px-wide bar with bar height ~self.height-48 (~150px on default panel) gives ~15px per band — visually smooth without per-pixel cost. Anti-aliasing is irrelevant for PZ's drawRect API (no GPU smoothing).

4. **Bar height computation uses `self.height - 48`** where 48 = 10 (top margin) + 24 (debuff slot height) + 10 (bottom margin) + 4 (gap above debuff row). This guarantees the bar never overlaps the debuff row regardless of panel size. Use `math.max(40, self.height - 48)` for safety on very short panels.

5. **Threshold tick orientation flips: was 1px-wide vertical lines spanning bar height; now 1px-tall horizontal lines spanning bar width.** Same data (4 ticks at 750/500/250/50), rotated 90°. Y position computed as `y + h - math.floor(h * (threshold/sanityMax))` so threshold 750 sits near the TOP (high sanity = top per fill direction), threshold 50 sits near the BOTTOM. The off-white tick color (`{r=0.9, g=0.9, b=0.9, a=1}`) stays.

```lua
-- ── Sanity bar: vertical orientation, always-on gradient, track-mask fill ────
-- GAP-01/GAP-02 closure (Plan 01.1-04). Amends D-05 and D-21 visual contract.
--
-- Render passes (in this order):
--   1. Gradient backdrop: ALWAYS render the full bar as 10 horizontal bands
--      lerping green->yellow->red from TOP to BOTTOM. The gradient is permanent
--      and never changes shape with sanity. (GAP-02 fix.)
--   2. Track mask: paint the empty TOP portion (height = (1 - pct) * h) with
--      BAR_TRACK dark grey. This hides the gradient where sanity is "drained."
--      The remaining bottom portion (height = pct * h) shows the gradient.
--      This is the FILL: bottom-up, anchored at the red end. (GAP-02 fix.)
--   3. Threshold ticks: 4 horizontal 1px lines spanning the bar's full width
--      at proportional Y positions along the bar. Higher threshold = nearer top
--      (since high sanity = top of bar). Always visible, drawn OVER the
--      track-mask so they remain visible in empty portions. (D-07 preserved.)
--   4. Border: drawn last so it sits over everything.
--
-- The function accepts arbitrary x/y/w/h so it works regardless of where the
-- caller positions the bar. Callers (only :render in this file) pass the
-- right-edge column coordinates (Task 2 sets barX = self.width - 14 - 10 etc.).
function SanityPanel:drawSanityBar(x, y, w, h, sanity, sanityMax)
    local pct = sanity / sanityMax    -- 0..1
    if pct < 0 then pct = 0 end
    if pct > 1 then pct = 1 end

    -- ── Pass 1: Always-on gradient backdrop (10 bands, GREEN at TOP -> RED at BOTTOM) ──
    local BAND_COUNT = 10
    local bandH = h / BAND_COUNT
    for band = 1, BAND_COUNT do
        local bandPct = (band - 0.5) / BAND_COUNT   -- 0..1, top->bottom centroid
        local r, g, b
        if bandPct < 0.5 then
            -- green to yellow: r climbs 0->1, g stays 1
            local t = bandPct / 0.5
            r, g, b = t, 1.0, 0.0
        else
            -- yellow to red: r stays 1, g drops 1->0
            local t = (bandPct - 0.5) / 0.5
            r, g, b = 1.0, 1.0 - t, 0.0
        end
        -- Use ceil for the last band to avoid sub-pixel gap at the bottom edge
        local bandY = y + math.floor((band - 1) * bandH)
        local thisBandH = (band == BAND_COUNT) and (y + h - bandY) or math.ceil(bandH)
        self:drawRect(x, bandY, w, thisBandH, 1, r, g, b)
    end

    -- ── Pass 2: Track mask over the empty TOP portion (fill from BOTTOM) ──
    -- emptyH = how much of the top is "drained." pct=1 -> emptyH=0 (full bar visible);
    -- pct=0 -> emptyH=h (no gradient visible, only dark track); pct=0.5 -> emptyH=h/2.
    local emptyH = math.floor(h * (1 - pct))
    if emptyH > 0 then
        self:drawRect(x, y, w, emptyH, 1, 0.1, 0.1, 0.1)   -- BAR_TRACK dark grey
    end
```
