## Decisions Made

1. **Insertion point** — chose to insert between `DEBUFF_SLOT_COUNT` and `print(...)` rather than after `print(...)`. Two reasons: (a) the loader-announcement print is a logical end-of-file marker; constants logically precede it; (b) keeps the print line as the last line of the file, matching the existing convention readers rely on.
2. **Expression form** — used `10 * 1000` (matching PLAN spec verbatim) instead of `10000`. Self-documents the 10-second intent without a comment-line reread.
3. **`stable` deliberately absent from STAGE_DESCENT_KEY** — recovery UP to stable is not a descent and should not bump a counter. Phase 2's transition handler will compute `STAGE_DESCENT_KEY[newStageKey]` and rely on the nil-return for stable to skip the bump. Documented inline in the comment block.
4. **LOG_MAX_ENTRIES kept** — Phase 01.2 supersedes D-11/D-12/D-15 (the chronological log system) but the migration approach (D-27) leaves the constant defined as harmless dead code rather than removing it. This avoids touching anything in the file beyond appending new content. Future cleanup phase can purge.
5. **Used `--no-verify` on the commit** — required by parallel-execution context (parallel Plan 01 executor running concurrently; pre-commit hook contention avoided).
