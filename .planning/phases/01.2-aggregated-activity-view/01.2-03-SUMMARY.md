# Phase 01.2 Plan 03: bumpCounter API + Listbox Surgery Summary

Replaced the Phase 01.1 SanityTraits.log/ISScrollingListBox event-log surface with the Phase 01.2 SanityTraits.bumpCounter dotted-path API and shifted the numeric readout from `"%d / %d (%d%%)"` to `"%d%%"` per D-26 — all in one `4_SanityTraits_Panel.lua` file, four atomic commits, every Phase 01.1 GAP-01..06 closure preserved verbatim.

## What Was Built

### Task 1 — Logger surface swap
- Deleted `function SanityTraits.log(category, text, delta)` and the `local VALID_LOG_CATEGORIES = { kill=true, stage=true, trait=true, recovery=true }` whitelist table.
- Deleted the `[SanityTraits] Panel loader: SanityTraits.log ready` load-receipt print statement.
- Inserted `function SanityTraits.bumpCounter(path, delta)` with:
  - Empty/nil path defensive guard (`if not path or path == "" then return end` — Risk 3 mitigation).
  - Live-player guard (`if not player then return end`).
  - ModData/counters seeded guard (`if not md.SanityTraits or not md.SanityTraits.counters then return end`).
  - Dotted-path walker via `string.gmatch(path, "[^.]+")` with intermediate-table auto-vivify.
  - Leaf cell auto-vivify on first call (`if cell[lastSeg] == nil then cell[lastSeg] = { count = 0 } end`).
  - `leaf.count = (leaf.count or 0) + 1` increment.
  - `leaf.touchedAt = getTimestampMs()` real-time wall-clock stamp (runtime-only field; ModData migration zeros it on every load per Pitfall 4).
  - `seenAt` deliberately NOT set here — it is render-layer-only.
  - Console.txt receipt: `[SanityTraits] counter[<path>] count=<N>` with optional `(delta=<d>)` suffix.
- New load-receipt: `[SanityTraits] Panel loader: SanityTraits.bumpCounter ready`.
- Updated top-of-file comment header to reference Phase 01.2 / Plan 03 + RESEARCH Pattern 3.

### Task 2 — Listbox surface teardown
- `:new` — removed `o.lastLogCount = -1` cache field (the `lastAppliedCount = -1` debuff cache is preserved verbatim).
- `:createChildren` — removed the entire 25-line ISScrollingListBox setup block: geometry locals (`logX`, `logY`, `logH`, `logW`), the `ISScrollingListBox:new` instantiation, `:initialise()`, `:instantiate()`, `:setFont(UIFont.Small, 4)`, `backgroundColor` opaque-grey assignment, `drawBorder = true` flag, and `self:addChild(self.eventLog)`.
- `:reflowLayout` — removed the `if self.eventLog then ... end` geometry-recompute block; the debuffSlots Y reflow loop (`for i = 1, SanityTraits.DEBUFF_SLOT_COUNT do ... self.debuffSlots[i]:setY(rowY)`) is preserved verbatim.
- Updated `:reflowLayout` and `:setWidth` doc-comments to reflect listbox absence; method signatures and bodies preserved.

### Task 3 — Listbox helpers deletion
- Deleted `local function formatLogEntryDisplay(entry)` (Phase 01.1 listbox label format with `D{day} {HH:MM} - {text} ({signedDelta})` time prefix).
- Deleted `function SanityPanel:refreshLogList(md)` (Pitfall-5-cached listbox rebuilder).
- Removed the `"No events yet."` empty-state placeholder string.
- The `local function getTraitTextureSafe(traitId)` helper (Critical Correction 1) and `function SanityPanel:refreshDebuffRow(md)` (D-16..D-19) are preserved verbatim — both use `TraitFactory.getTrait(traitId)` and `slot:setMouseOverText(tipText)` exactly as Phase 01.1 shipped them.

### Task 4 — :render numeric format + call cleanup
- Numeric readout: `string.format("%d / %d (%d%%)", sanity, sanityMax, pct)` → `string.format("%d%%", pct)`.
  - Internal `md.sanity` is unchanged (still raw 0-1000 integer).
  - `pct = math.floor((sanity / sanityMax) * 100)` math is unchanged — only the format string changed.
  - F11 console invariant preserved: `getPlayer():getModData().SanityTraits.sanity` returns 1000 (raw), not 100 (pct).
  - Updated comment block to document D-26 supersedes D-06.
- Removed `self:refreshLogList(md)` call from `:render`.
- Preserved `self:refreshDebuffRow(md)` call.
- Inserted explanatory comment block where the listbox refresh used to be — Plan 04's `self:renderCounterTree(md)` call site is reserved here.
- Header divider drawRect at `(10, 50, self.width - 20 - (barW + 18), 1, 1, 0.4, 0.4, 0.4)` preserved verbatim per plan instruction; surrounding doc-comment updated for accuracy ("content area" instead of "event log").
