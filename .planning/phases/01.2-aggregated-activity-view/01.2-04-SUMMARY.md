## Recency-Fade Algorithm (D-28 Detail)

```
on each render frame, for each counter cell:
  if cell.touchedAt > cell.seenAt:
    cell.seenAt = now            -- D-28 render-layer stamp
  age = now - cell.seenAt
  t = clamp(age / FADE_WINDOW_MS, 0, 1)   -- FADE_WINDOW_MS = 10000ms
  color = lerp(WHITE, MID_GREY, t)
  drawText(cellText, x, y, color.r, color.g, color.b, alpha=1)
```

**Three timing scenarios all handled** (verified by D-28 + Plan 01 clearTransientFields):

1. **Tab open during increment** → bumpCounter sets `touchedAt = now`. Next render: `touchedAt > seenAt (0)` → stamp `seenAt = now` → color WHITE. Subsequent frames lerp toward grey.
2. **Tab closed during increment** → bumpCounter sets `touchedAt`. No render. When tab opens, first render: `touchedAt > seenAt` → stamp `seenAt = now-on-open` → color WHITE at moment of opening; fades from there.
3. **Save/reload mid-fade** → `getTimestampMs()` is wall-clock since Java process launch (NOT save-stable). Plan 01's `clearTransientFields` on `OnCreatePlayer` re-zeroes both `touchedAt` and `seenAt` on every load. First render after load: `touchedAt (0) <= seenAt (0)` → no stamp → `age = now - 0` is huge → color fully MID_GREY. Persisted counters appear stale ("not new") on reopen — exact correct semantics per CONTEXT.md persistence rule.

## Pitfall Mitigations Applied

- **Pitfall 1 (pairs() ordering non-determinism):** `STAGE_DESCENT_ORDER` fixed list iterated via `ipairs`. The traitsAcquired subcategory uses `pairs` only to build a list, then `table.sort` for deterministic ordering by `touchedAt desc`.
- **Pitfall 3 (slot-index ordering):** `renderCounterTree` uses an emit-time `rowIdx` counter (incremented per emitted row) — the build sequence IS the visible-rows ordering. No fixed-slot index that could orphan a row when an earlier row is hidden.
- **Pitfall 4 (cross-restart timing):** Mitigation handled in Plan 01 (`clearTransientFields`). Plan 04's render layer trusts re-zeroed transient fields and produces fully-grey on first frame after load (no phantom flash).
