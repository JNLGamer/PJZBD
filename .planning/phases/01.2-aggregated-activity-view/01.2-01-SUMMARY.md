## Decisions Made

None beyond plan — followed CONTEXT D-26..D-31 + RESEARCH Pattern 6 + Pitfall 4 verbatim. Notable plan-mandated choices encoded:

- **Persistence rule (D-28 appendix):** Per-cell schema seeded as `{ count = 0 }` only — no `touchedAt = 0, seenAt = 0` literal keys at seed time. This is the exact RESEARCH Pattern 2 recommendation: runtime-only fields default to 0 via `or 0` at render time, so seeding them isn't necessary; absence + render-time defaulting + walker re-zero on load is bulletproof.
- **Walker placement:** After both new-char and loaded-save branches resolve (outside the `if/else`), so it runs unconditionally on every fire. Defensive `if md.SanityTraits and md.SanityTraits.counters then` guard prevents NPE on degenerate paths.
- **Walker leaf detection:** `t.count ~= nil` distinguishes leaf cells (have a count integer) from intermediate nodes (stageDescents, traitsAcquired) — recurses on intermediates. Handles dynamic `traitsAcquired` keys correctly: empty table → for-pairs is no-op; populated by Phase 2 with `["sanity_xyz"] = { count = N, touchedAt = ..., seenAt = ... }` → walker descends, hits count, zeroes timestamps.
