# Mod Structure (Build 42)

Source of truth: [`wiki_dump/Mod structure - PZwiki.txt`](wiki_dump/Mod%20structure%20-%20PZwiki.txt) (PZwiki, last updated for 42.14.0). Verified against installed workshop mods on this machine.

> **The single most important rule:** in B42, *at least one* `42/`/`42.X/`/`common/` folder is required for the mod to be recognized in-game. Without it, B42 silently treats the mod as B41-only and hides it from the Mods menu.

## The two recognized mod locations

PZ scans for local mods in two directories under the cache folder (`%UserProfile%/Zomboid/`):

| Location | When to use | Notes |
|----------|-------------|-------|
| `Zomboid/mods/` | Manual installs | Wiki: **"Not recommended for mod development."** |
| `Zomboid/Workshop/<MyMod>/Contents/mods/` | Active development + Workshop publishing | Recommended; required for the in-game uploader |

If the same mod ID exists in both, they clash and silently overwrite each other — keep only one copy.

There is also `Steam/steamapps/workshop/content/108600/<workshopID>/` for downloaded subscriptions; never edit those.

## Canonical B42 mod folder layout (development)

```
%UserProfile%/Zomboid/Workshop/MyMod/
├── workshop.txt                       ← auto-generated by in-game uploader
├── preview.png                        ← 256x256 Workshop thumbnail
└── Contents/
    └── mods/
        └── MyMod1/
            ├── mod.info               ← root-level mod.info (recommended, see note)
            ├── poster.png             ← optional, in-game mod-list thumbnail
            ├── icon.png               ← optional
            ├── common/                ← MANDATORY (can be empty), or…
            │   └── media/             ← shared assets across versions
            └── 42/                    ← …a version folder. AT LEAST ONE OF common/ OR 42/ IS MANDATORY
                ├── mod.info           ← version-specific mod.info (overrides root)
                ├── poster.png
                └── media/
                    ├── lua/
                    │   ├── client/
                    │   ├── server/
                    │   └── shared/
                    └── scripts/
```

**Note on root vs version-folder mod.info:** the wiki shows mod.info at the root *and* inside the version folder in its "full example." In practice, the version folder's mod.info is what B42 reads; the root one is for B41 compatibility when mixing structures.

## Minimal layout for a B42-only Zomboid/mods/ install

If you're putting a mod directly in `Zomboid/mods/<ModName>/` (faster for development, doesn't need Workshop scaffolding):

```
Zomboid/mods/MyMod/
├── common/                ← empty marker — required for B42 detection
└── 42/
    ├── mod.info
    └── media/
        └── lua/client/
            └── *.lua
```

The `common/` folder must exist even if empty — it's the marker that tells PZ "this mod knows about the B42 structure." Without it, even a properly-configured `42/` folder may not be enough.

## Version folder naming and matching

Version folders match by major version, with the minor stripped:

| Folder name | Treated as | Loads for game version |
|-------------|-----------|------------------------|
| `42` | `42.0` | any 42.x |
| `42.12` | `42.12` | 42.12.x and above (until a higher version folder exists) |
| `42.0.5` | `42.0` (minor stripped) | any 42.x |
| `43.5.1` | `43.5` | 43.5.x and above |

Loading order:
1. `common/` loads first
2. The closest version folder ≤ current game version loads next, **overwriting** any same-named files from `common/`

You don't need a folder for every version. Use multiple version folders only when a mod must ship different code for different game versions.

## Mixing B41 and B42 structures

You can ship a single mod that works for both builds because the structures don't overlap:

```
MyMod1/
├── common/         ← B42 marker, mandatory for B42 detection
├── 42/             ← B42 content
│   ├── mod.info
│   └── media/...
└── media/          ← B41 content (B41 reads this, B42 ignores it)
    └── ...
```

The B41 path is `media/...`; the B42 path is `42/media/...` — one folder deeper. They never collide.

## media/ subfolder reference

Inside any `42/media/` (or `common/media/`):

| Subfolder | Contains |
|-----------|----------|
| `lua/client/` | Client-only Lua (UI, input, anything player-local) |
| `lua/server/` | Server-only Lua (economy, world spawning, MP authoritative logic) |
| `lua/shared/` | Lua loaded on both sides — items, recipes, traits, events |
| `lua/shared/Translate/` | Translation files (subfolder per language) |
| `scripts/` | `.txt` script definitions (items, recipes, professions, traits) |
| `clothing/` | Clothing item XML + `clothing.xml` outfit manager |
| `models_X/` | `.x` (DirectX, deprecated) or `.fbx` model files |
| `textures/` | PNG textures. Item icons must be named `item_<name>.png` |
| `texturepacks/` | Tile sprite packs (use Mapping tools to unpack) |
| `ui/` | UI element PNGs |
| `sound/` | `.ogg` or `.wav` files (`.bank` files cannot be loaded from mods) |
| `anims_X/` | Animation files |
| `AnimSets/` | Animation triggers and parameter definitions |
| `maps/` | Map files and assets |

## Lua load order within a folder

PZ loads `.lua` files alphabetically. Convention: prefix with a number (`1_Init.lua`, `2_ModData.lua`, `3_Events.lua`) to enforce load order. `lua/shared/` loads before `lua/client/` and `lua/server/`.

## mod.info field reference

Required-or-recommended fields (verified against `Zomboid/Workshop/ModTemplate/Contents/mods/ModTemplate/mod.info` and confirmed-loading workshop mods on this machine):

```
name=Your Mod Display Name
id=YourModID                ← must be globally unique; used in save files and MP
description=Short text shown in the in-game Mods list.
modversion=1.0              ← your version string
pzversion=42                ← target PZ major version. Use 42 (not 42.0) for B42
versionMin=42.0.0           ← minimum game version the mod supports (recommended for B42)
poster=poster.png           ← optional, thumbnail in mod list
icon=icon.png               ← optional, smaller icon
url=https://...             ← optional, link shown in mod info panel
require=OtherModID          ← optional, mod must be loaded after these dependencies
incompatible=BadModID       ← optional, refuses to load if these are active
```

**`pzversion` rules verified by reverse-engineering the game source:**

- `pzversion=42` — works (matches major; this is what most workshop B42 mods use)
- `pzversion=42.13` — works (matches game 42.13.x and above)
- `pzversion=42.0` — does NOT work; `ModInfo:isAvailable()` returns false, mod hidden from menu
- Missing `pzversion` — the official `ModTemplate/mod.info` ships without it and works (probably whitelisted; do not rely on this)

**`Zomboid/mods/default.txt`** is a separate file that records *which* mods are enabled, not which exist. PZ rewrites it on save. Empty `mods { }` block means nothing is enabled, not that nothing is detected.

## Why mods don't appear in the Mods menu

Decision tree, in order of likelihood (B42, mod placed under `Zomboid/mods/<ModName>/`):

1. **No `42/`/`42.X/`/`common/` folder** → mod treated as B41-only, hidden as unsupported. **Most common cause.**
2. **`pzversion=42.0` (instead of `42`)** → `isAvailable()` false, hidden as unsupported.
3. **Mod is hidden because "Show unsupported mods" tickbox is off** → toggle it in the Mod selector to confirm.
4. **Folder name conflict / duplicate id** → if the same mod ID exists in `mods/`, `Workshop/`, and `steamapps/workshop/content/108600/`, they fight; only one wins. Remove duplicates.
5. **Case sensitivity (Linux/macOS only)** → Windows is forgiving. Use exact case (`common`, not `Common`).
6. **PZ caches the mods list across launches** → fully exit (not just main-menu back) and relaunch when iterating on mod.info.

## OS-specific notes

- **Linux/macOS** are case-sensitive on file/folder names. Use lowercase exactly: `common`, `media`, `lua`, `client`, etc.
- **Windows** is case-insensitive at the filesystem layer but PZ may still match case-sensitively in some paths — match the wiki examples exactly to be safe.
